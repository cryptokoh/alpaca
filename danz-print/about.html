About

The Story Behind Alpaca3D

A maker project born from curiosity, Claude Code, and a really fast printer.

It started with a simple question: what happens when you point an AI at a 3D printer and let it figure things out? Not a hypothetical research paper question. A real one, with real filament, real failed prints, and a very real Flashforge AD5M Pro sitting on a desk.

Alpaca3D is the answer. It is a pipeline that connects Claude Code (an AI reasoning engine from Anthropic) to a Flashforge AD5M Pro running Klipper firmware, bridged by OpenClaw -- an open-source translation layer that converts high-level intent into low-level machine instructions.

The idea is not to automate makers out of the loop. It is to give them better tools. Describe what you want in natural language. Let the AI handle the tedious parameter tuning, the retraction settings, the temperature profiles. You focus on the design. The machine handles the math.

Every line of code in this project was written in the open. Every failed print was documented. Every breakthrough was shared. That is the maker way.

Deep Dive

The Stack, Expanded

Each layer of the Alpaca3D pipeline, explained in detail.

🤖

Claude Code

The intelligence layer. Claude Code acts as the reasoning engine that interprets your natural language descriptions and translates them into actionable 3D printing parameters.

  • Natural language to G-code parameter mapping
  • Print failure root cause analysis
  • Material-specific profile optimization
  • Iterative design refinement through conversation
  • Real-time troubleshooting during active prints
🔌

OpenClaw

The translation layer. OpenClaw is the open-source bridge between AI-generated intent and printer firmware commands. Think of it as a universal adapter.

  • Bi-directional communication with Klipper firmware
  • Real-time telemetry streaming (temps, position, flow)
  • G-code validation and safety checks
  • Multi-printer orchestration support
  • REST API for external tool integration
⚙️

Flashforge AD5M Pro

The fabrication layer. A high-speed CoreXY printer modified with Klipper firmware for maximum control and print speed.

  • 600mm/s maximum print speed (CoreXY kinematics)
  • Automatic bed leveling with 25-point mesh
  • Filament runout detection and auto-pause
  • Network control via built-in WiFi
  • Camera-based first layer monitoring
Philosophy

What we believe

The principles that guide every decision in the Alpaca3D project.

🔓

Open Source

Every line of code is public. Fork it, break it, make it yours. No paywalls, no premium tiers, no vendor lock-in. The best ideas grow when they are shared freely.

🧠

AI-Augmented

Claude Code does not replace the maker -- it amplifies them. The AI handles parameter optimization, failure analysis, and the boring repetitive stuff. You handle the creative vision.

🔧

Maker-First

Built by makers, for makers. No gatekeeping, no credentialism. If you have a printer and an idea, you have everything you need to contribute to this project.

The Mascot

Why an alpaca?

Glad you asked.

Alpacas are calm, curious, and surprisingly practical. They do not rush. They observe. They process. And when they are ready, they produce something incredibly useful -- fiber that is softer than cashmere and warmer than wool.

That is basically what this project does. It takes raw ideas, processes them through layers of intelligence, and produces something tangible. Also, alpacas are famously good at spitting when provoked, which felt like an appropriate metaphor for debugging 3D printer firmware at 2 AM.

Plus, "Alpaca3D" is just a great name. Try saying it out loud. You will smile. That counts for something.
